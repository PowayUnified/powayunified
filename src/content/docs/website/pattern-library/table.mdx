**The Table Component** provides accessible, branded data tables for the Edlio CMS. Simply copy and paste any table style below directly into the WYSIWYG editor, then customize the content to fit your needs.

## Key Features

<CardGrid>
  <Card title="Multiple Header Styles" icon="list-format">
    Navy Alt, Dark Navy, PUSD Gold, and Light Blue header variants to match any content type or page context.
  </Card>
  <Card title="Accessible Markup" icon="approve-check-circle">
    Semantic `<table>`, `<thead>`, `<tbody>`, `<th>`, and `<td>` elements. Headers provide automatic context to screen readers.
  </Card>
  <Card title="Striping Patterns" icon="bars">
    Subtle light stripe (`#f8f9fa`) and strong contrast stripe (`#d3d3d3`) options keep rows scannable.
  </Card>
  <Card title="No Dependencies" icon="information">
    Pure HTML with inline CSS — works immediately in Edlio CMS with no external stylesheets required.
  </Card>
</CardGrid>

## Table Styles & When to Use

| Style | Header Color | Best Used For |
|---|---|---|
| **Basic (Alt Navy)** | `#3a57a8` + light stripe | General data, schedules, comparisons |
| **Striped (Gray)** | `#3a57a8` + gray stripe | Contact lists, directories, rosters |
| **Gold Header** | `#f0ad4e` + warm stripe | Events, calendars, announcements |
| **Light Blue Header** | `#54c8eb` + blue stripe | Resources, guides, reference lists |
| **Compact (Minimal)** | `#003366` border-bottom only | Dates, quick reference, key-value pairs |
| **Two-Column Rich** | `#3a57a8` + light stripe | Division contacts with multi-line cell content |

## Layout Options

| Option | How to Apply | Notes |
|---|---|---|
| **Full width** | `width: 100%` on `<table>` | Default for all variants |
| **Fixed columns** | `table-layout: fixed` on `<table>` | Prevents overflow; columns share width equally |
| **Custom column widths** | Add `width: XX%` to `<th>` or `<td>` | Column widths must total 100% |
| **Rich cell content** | Use `<strong>`, `<br>`, `<a href="">` inside `<td>` | See Two-Column Rich variant |
| **Center-aligned text** | `text-align: center` on `<th>` or `<td>` | Override `text-align: left` as needed |

## Customization Guide

<Steps>
1. **Add rows** — Copy an existing `<tr>...</tr>` block from `<tbody>` and paste it below the last row. Maintain the correct alternating `background-color` to preserve striping.

2. **Remove rows** — Delete the entire `<tr>...</tr>` block for any row you don't need.

3. **Add columns** — Add a `<th>` in `<thead>` and a matching `<td>` in every `<tbody>` row. Keep `<th>` and `<td>` counts consistent across all rows.

4. **Remove columns** — Delete the corresponding `<th>` and all matching `<td>` elements from every row.

5. **Change header color** — Replace the `background-color` value in the `<tr style="...">` inside `<thead>`. Also update the `border` color on `<th>` elements to match.

6. **Adjust column widths** — Add `width: XX%;` to individual `<th>` style attributes. Widths should total 100%.

7. **Add rich content** — Use `<strong>`, `<br>`, and `<a href="">` tags directly inside `<td>` cells for multi-line or linked content.

8. **Adjust text alignment** — Change `text-align: left` to `center` or `right` on any `<th>` or `<td>` as needed.
</Steps>

## Accessibility

<Aside type="note" title="Accessibility Features">
- Semantic `<table>`, `<thead>`, `<tbody>`, `<th>`, and `<td>` structure for correct screen reader interpretation
- `<th>` header elements automatically provide column context — no `aria-label` needed for standard data tables
- All color combinations meet **WCAG AA** contrast standards
- Minimum `font-size: 14px` ensures text legibility at all sizes
- `width: 100%` and `table-layout: fixed` ensure the table scales within its container without horizontal overflow
</Aside>

## Best Practices

<Aside type="tip" title="Do's">
- Use tables for **tabular data only** — not for page layout
- Limit tables to **3–5 columns** for readability, especially on mobile
- Use **descriptive column headers** that clearly identify the data below
- **Sort rows logically** — alphabetically or by date — for easy scanning
- Use the same table style **consistently** throughout a page
- Use **rich content** (`<strong>`, `<br>`, `<a>`) inside cells to add detail without adding columns
</Aside>

<Aside type="caution" title="Don'ts">
- Don't use tables to control page layout or positioning
- Don't mix multiple header color styles on a single page
- Don't omit `<thead>` — it is required for screen reader column context
- Don't use very small font sizes below `14px` inside cells
</Aside>
