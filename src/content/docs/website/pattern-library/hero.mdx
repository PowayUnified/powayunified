import { Tabs, TabItem, Aside, Card, CardGrid, Steps } from '@astrojs/starlight/components';

**The Hero Component** creates impactful page introductions that appear immediately after the Edlio CMS page title to welcome users, set context, and guide them toward key actions. Available in text-only and image background variants, each with optional button support.

## Key Features

<CardGrid>
  <Card title="7 Text-Only Styles" icon="list-format">
    Welcome, Department, Page Intro, Centered, Minimal, Info Callout, and Announcement variants — each with or without button support.
  </Card>
  <Card title="5 Image Background Styles" icon="puzzle">
    Dark overlay, light overlay, and directional gradient overlays for high-impact visual storytelling with background images.
  </Card>
  <Card title="Flexible Button Support" icon="approve-check-circle">
    Single or dual buttons using Navy, Light Blue, Gold, Outline, and Alt Navy styles — composable with any hero type.
  </Card>
  <Card title="No Dependencies" icon="information">
    Pure HTML with inline CSS — paste directly into Edlio WYSIWYG with no external stylesheets required.
  </Card>
</CardGrid>

## Hero Types Overview

| Type | Background | Best Used For |
|---|---|---|
| **Welcome Hero** | Navy gradient (`#003366` → `#3a57a8`) | District home page, school landing pages |
| **Department Hero** | White + Light Blue left border | Department overview, service pages |
| **Page Intro** | Gold gradient (`#f0ad4e` → `#ffc870`) | Information pages, general school pages |
| **Centered Text** | Solid Navy (`#003366`) | Community messages, welcome statements |
| **Minimal** | Light gray (`#f8f9fa`) + border | Quick-access pages, resource hubs |
| **Info Callout** | Light Blue (`#e8f7fc`) + icon | Process pages, enrollment, how-to guides |
| **Announcement** | Alt Navy gray (`#e6eaf0`) + gold badge | Time-sensitive notices, important updates |
| **Image: Dark Overlay** | Background image + navy rgba overlay | High-impact welcome, visual storytelling |
| **Image: Light Overlay** | Background image + white rgba overlay | Bright, content-forward visual pages |
| **Image: Directional Gradient** | Background image + directional gradient | Department pages, left-aligned layouts |

## Customization Guide

<Steps>
1. **Change subtitle text** — Update the first `<p>` tag content. Keep to 1–2 sentences maximum.

2. **Change description text** — Update the second `<p>` tag content. Keep to 2–3 sentences maximum.

3. **Add a button** — Copy a button `<a>` tag from the Button Styles table above and paste it after the last `<p>` tag inside the hero `<div>`.

4. **Change button text** — Replace the text between the opening and closing `<a>` tags with your action-oriented label (e.g., "Enroll Now", "Get Started", "View Resources").

5. **Change button URL** — Update the `href="https://www.example.com"` attribute with your actual destination URL.

6. **Change button color** — Swap the `background-color`, `color`, and `border` values using the Button Styles table above.

7. **Add a second button** — Wrap both `<a>` buttons in a flex container: `<div style="display: flex; gap: 12px; justify-content: center; flex-wrap: wrap;">...</div>`. See Page Intro and Light Overlay examples.

8. **Change the icon** (Info Callout) — Replace the `ℹ` Unicode character in the `<span>` with another symbol (e.g., `●`, `★`, `✓`, `⚠`).

9. **Adjust spacing** — Modify `padding` values (e.g., `30px`, `40px`) on the outer `<div>` for more or less vertical space.

10. **Remove elements** — Delete an entire `<p>` or `<a>` tag for a simpler, stripped-down version of any hero.

11. **Change background image** — Replace `YOUR-IMAGE-URL-HERE` with your image URL. Use high-resolution images (1920×800px minimum).

12. **Adjust overlay darkness** — Change the `rgba()` opacity value: `0.5` = 50% dark, `0.75` = 75% dark, `0.85` = 85% light, `0.9` = 90% dark.

13. **Change overlay gradient direction** — Replace `linear-gradient(rgba(...), rgba(...))` with `linear-gradient(to top, ...)` for bottom-to-top or `linear-gradient(to right, ...)` for left-to-right.

14. **Change text alignment for image heroes** — Modify `justify-content` (center / flex-start / flex-end) and `align-items` (center / flex-start / flex-end) on the outer `<div>`.

15. **Adjust minimum height** — Change `min-height` (e.g., `300px`, `400px`, `500px`) for taller or shorter image heroes.
</Steps>

## Accessibility

<Aside type="note" title="Accessibility Features">
- No `<h1>` tags — Edlio CMS provides the page title; heroes use `<p>` tags to maintain correct heading hierarchy
- `aria-hidden="true"` on decorative icon `<span>` elements hides them from screen readers
- `role="img"` and `aria-label` on image background heroes provide context to screen reader users
- `text-shadow` on image backgrounds ensures text remains readable over varied image content
- All color combinations meet **WCAG 2.1 AA** contrast standards (4.5:1 minimum ratio for text)
- Dark and light overlays ensure text readability regardless of the underlying image
- Button `<a>` tags include descriptive, action-oriented text for clear link context
- Flexbox layouts are keyboard-accessible with proper focus states
</Aside>

## Best Practices

<Aside type="tip" title="Do's">
- Place the hero **immediately after the Edlio CMS page title** — it should be the first content element
- Use **one hero per page** for maximum impact
- Keep subtitle text to **1–2 sentences maximum**
- Keep description text to **2–3 sentences maximum**
- Use **action-oriented button text** — "Get Started", "Learn More", "Enroll Now", "View Resources"
- **Match the hero type to page context** — Welcome for district pages, Department for department pages, Info Callout for process pages, Announcement for urgent notices
- **Test on mobile devices** before publishing to ensure proper rendering
- **Update all button URLs** from `https://www.example.com` before publishing
- Use **optimized, compressed images** for image background heroes
</Aside>

<Aside type="caution" title="Don'ts">
- Don't add `<h1>`, `<h2>`, or `<h3>` tags inside hero components — the CMS manages heading hierarchy
- Don't use more than **two buttons** in a single hero — it dilutes the call to action
- Don't use image background heroes with low-contrast or busy images — always verify text readability
- Don't use `YOUR-IMAGE-URL-HERE` placeholder in published content
- Don't use heroes for layout structuring — they are page introductions only, not content containers
</Aside>
