**The Table of Contents Component** provides navigation for long-form content, helping users quickly jump to specific sections of a page. Enhances user experience by making lengthy pages easier to navigate and scan.

## Key Features

<CardGrid>
  <Card title="Sticky Positioning" icon="rocket">
    Stays visible as users scroll down long pages. Automatically falls back to static positioning on mobile via a responsive media query.
  </Card>
  <Card title="Hierarchical Support" icon="list-format">
    Nested subsections with `padding-left: 24px` indentation clearly convey page structure at a glance.
  </Card>
  <Card title="4 Color Themes" icon="seti:css">
    Default (white), Navy, Light Blue, and Gold Accent variants match any page context or department branding.
  </Card>
  <Card title="Accessible Markup" icon="approve-check-circle">
    Semantic `<nav>` with `aria-label`, `<h2>` heading, and `<ul>` list structure for full screen reader and keyboard support.
  </Card>
</CardGrid>

## Variants Overview

| Variant | Background | Border | Best Used For |
|---|---|---|---|
| **Default** | White `#ffffff` | Light Blue `#54c8eb` | General long-form pages |
| **Navy** | Navy `#003366` | Light Blue `#54c8eb` | Pages using dark hero or navy sections |
| **Light Blue** | Light Blue tint `#e8f7fc` | Navy `#003366` | Information, resource, or enrollment pages |
| **Gold Accent** | Cream `#fef9f0` | Gold `#f0ad4e` | Announcements, event pages, spotlights |
| **Hierarchical** | White `#ffffff` | Light Blue `#54c8eb` | Complex pages with nested subsections |
| **Compact (On This Page)** | Light Blue tint `#e8f7fc` | Light Blue `#54c8eb` | Shorter pages; non-sticky inline use |

## Anchor Link Setup

The TOC is only functional when each `href` points to a matching `id` on a heading in your page content. Add `id` attributes to your section headings after pasting the TOC.

```html
<!-- In your TOC -->
<a href="#introduction">Introduction</a>

<!-- In your page content -->
<h2 id="introduction">Introduction</h2>
<p>Your introduction content here...</p>

<!-- Another section -->
<h2 id="getting-started">Getting Started</h2>
<p>Your getting started content here...</p>
```

## Customization Guide

<Steps>
1. **Change the heading text** — Update the text inside the `<h2>` tag. Common alternatives: `"Contents"`, `"On This Page"`, `"In This Guide"`.

2. **Add a link item** — Copy an existing `<li>...</li>` block, paste it after the last `<li>`, and update the `href` and link text.

3. **Remove a link item** — Delete the entire `<li>...</li>` block for any section you don't need.

4. **Update anchor links** — Change `href="#section-1"` on each `<a>` to match the `id` attribute on the corresponding section heading in your page (e.g., `href="#enrollment-dates"`).

5. **Add nested subsections** — Copy a top-level `<li>` and add `padding-left: 24px;` to the `<a>` style. Use a numbered label like `2.1 Prerequisites` for clarity.

6. **Remove sticky behavior** — Delete `position: sticky; top: 12px;` from the `<nav>` style attribute. The TOC will render inline with the page flow.

7. **Change the accent color** — Replace the border and hover color values in the `<style>` block and on the `<h2>` `border-bottom`. Use the PUSD Brand Colors table below for available values.

8. **Adjust spacing** — Modify `padding` on the `<nav>` (e.g., `16px 20px` for more space) and `padding` on `<a>` elements (e.g., `8px 10px`) for tighter or looser link rows.
</Steps>

## Accessibility

<Aside type="note" title="Accessibility Features">
- `<nav>` element with a unique `aria-label` identifies the TOC as a landmark region for screen readers
- `<h2>` inside the `<nav>` provides a visible and programmatic label for the block
- `<ul>` and `<li>` list structure ensures screen readers enumerate TOC items correctly
- Each `<a>` has `outline: 2px solid` on `:focus` for keyboard navigation visibility
- The `border-left` indicator on `:hover` and `:active` provides clear interaction feedback
- Descriptive link text (not "click here") gives context to screen reader users navigating by links
- `transition: all 0.3s ease` on links respects fluid interaction without causing disorientation
</Aside>

## Best Practices

<Aside type="tip" title="Do's">
- Use a TOC for pages with **3 or more major sections** or 1,000+ words of content
- **Place the TOC near the top** of the page content, before the first section heading
- Keep link labels **3–6 words** for easy scanning
- **Match the TOC order** exactly to the page section order — don't rearrange
- Use **numbered labels** (`1.`, `2.1`, etc.) for hierarchical TOCs to reinforce structure
- **Test every anchor link** after pasting to confirm each `href` resolves to a valid section `id`
- Use the **Compact (non-sticky) variant** for shorter pages or when sticky behavior isn't appropriate
- **Use one TOC style consistently** across similar pages within a site section
</Aside>

<Aside type="caution" title="Don'ts">
- Don't use a TOC for pages with fewer than 3 sections — it adds noise without value
- Don't copy only the `<nav>` block without the `<style>` block — hover and focus states will be missing
- Don't use placeholder `href="#section-1"` links on published pages — update all anchors before publishing
- Don't nest more than **2 levels deep** — deeper nesting becomes difficult to scan
- Don't modify the `aria-label` value without also updating the matching CSS selectors in the `<style>` block — the hover styles will break
</Aside>
