**The Card component provides flexible, reusable containers for content like images, text, and links.**

Cards are versatile UI elements that help organize information in a visually appealing, scannable format. They're perfect for displaying resources, news items, staff profiles, programs, and any content that benefits from a contained, structured presentation.

## Key Features

<CardGrid>
  <Card title="Multiple Card Styles" icon="list-format">
    Image, Icon, Horizontal, Featured, Simple, and Button CTA variants to match any content need.
  </Card>
  <Card title="Responsive Grid Layouts" icon="laptop">
    Automatically adapts from multiple columns to a single column on mobile using CSS Grid `auto-fit`.
  </Card>
  <Card title="Accessible Markup" icon="approve-check-circle">
    Semantic `<article>` elements with proper ARIA labels, alt text requirements, and keyboard navigation.
  </Card>
  <Card title="No Dependencies" icon="information">
    Pure HTML and inline CSS — no JavaScript required. Paste directly into the Edlio CMS WYSIWYG editor.
  </Card>
</CardGrid>

## Card Types

### Usage Guide

| Card Type | Best Used For | Key Features |
|---|---|---|
| **Image Card** | General content, news, programs, resources | Standard layout with image on top |
| **Icon Card** | Services, features, categories | Centered design with circular icon background |
| **Horizontal Card** | Staff profiles, news items, detailed content | Side-by-side image and text layout |
| **Featured Card** | Important announcements, highlighted programs | Navy gradient header with optional gold badge |
| **Simple Card** | Quick links, text-heavy content, lists | No image, minimal styling with border |
| **Button Card** | Call-to-action focused content | Includes a prominent button for a clear next step |

## Customization Guide

<Steps>
1. **Replace image URLs** — Update the `src` attribute on `<img>` tags with your actual image path.

2. **Update links** — Change the `href` attribute on `<a>` tags to your target page URL.

3. **Update ARIA labels** — Edit the `aria-label` attribute text to match your actual card title for screen reader accuracy.

4. **Modify titles** — Edit the text inside `<h3>` tags.

5. **Edit descriptions** — Update the text inside `<p>` tags.

6. **Adjust grid columns** — Change `minmax(280px, 1fr)` to control the minimum card width. A larger value means fewer columns; a smaller value allows more.

7. **Change aspect ratios** — Modify `aspect-ratio: 4/3` to `16/9` for widescreen images or `1/1` for square images.

8. **Add or remove elements** — Delete the `<img>` wrapper block to remove the image, or omit the button `<a>` tag on Button Cards to simplify.
</Steps>

## Accessibility

<Aside type="note" title="Accessibility Features">
- Semantic `<article>` elements provide proper document structure for screen readers
- Descriptive `aria-label` attributes on links convey destination context beyond visible text
- All `<img>` elements require meaningful `alt` text — describe what is shown, not just the filename
- Proper heading hierarchy using `<h3>` inside each card
- All color combinations meet **WCAG AA** contrast ratios
- All links are keyboard accessible and focusable
- SVG icons include `aria-hidden="true"` to prevent redundant screen reader announcements
</Aside>

## Best Practices

<Aside type="tip" title="Do's">
- Use **high-quality images** at appropriate dimensions (recommended: 400×300px or larger for 4:3 cards)
- Keep card **titles concise** — 3 to 7 words
- Limit **descriptions** to 1–2 sentences
- Always add `target="_blank"` and `rel="noopener noreferrer"` for all external links
- Provide **meaningful alt text** that describes the image content
- Test cards on **mobile devices** to confirm proper scaling and stacking
- Use the **same card style** for related content within a single grid
- Limit grids to **3–4 cards per row** maximum to avoid overcrowding
</Aside>

<Aside type="caution" title="Don'ts">
- Don't mix too many different card styles within a single grid section
- Avoid using very long card titles or multi-paragraph descriptions
- Don't omit `alt` text on images — use `alt=""` only for purely decorative images
- Don't use the Featured Card style for routine content — reserve it for important announcements
</Aside>
