**Badges** are small, inline labels used to highlight key information, categorize content, or indicate status. They provide quick visual identification and improve content scannability.

This component is designed for easy copy-paste use in the Edlio CMS WYSIWYG editor with zero dependencies.

## Customization Guide

### Color Reference

| Color Name | HEX | Text Color | Use For |
|---|---|---|---|
| PUSD Navy | `#003366` | `#ffffff` | Primary brand, important categories |
| PUSD Light Blue | `#54c8eb` | `#003366` | Secondary brand, informational |
| PUSD Gold | `#f0ad4e` | `#ffffff` | Accent, warnings, highlights |
| Alt 1 (Light Gray-Blue) | `#dce0ef` | `#003366` | Subtle emphasis |
| Alt 2 (Medium Blue) | `#a5afd6` | `#003366` | Moderate emphasis |
| Alt 3 (Deep Blue) | `#3a57a8` | `#ffffff` | Strong emphasis |
| Success (Green) | `#4caf50` | `#ffffff` | Positive status |
| Error (Red) | `#d32f2f` | `#ffffff` | Negative status |

### Size Reference

| Size | `padding` | `font-size` | `border-radius` |
|---|---|---|---|
| Small | `2px 8px` | `12px` | `10px` |
| Medium (Default) | `4px 12px` | `14px` | `12px` |
| Large | `6px 16px` | `16px` | `14px` |

<Aside type="note" title="Outline Size Adjustment">
Outline badges use `padding: 3px 11px` (1px less on each side) to compensate for the `border: 2px solid` and maintain the same visual height as filled badges.
</Aside>

### Quick Customization Tips

<Steps>
1. **Change the text** — Replace the content between the `<span>` tags.

2. **Change the color** — Update `background-color` and `color` values using the HEX codes from the color reference table above.

3. **Make it an outline badge** — Set `background-color: transparent`, reduce padding by 1px on each side, and add `border: 2px solid [color]`.

4. **Make it a pill shape** — Change `border-radius` to `999px`.

5. **Adjust the size** — Modify `padding` and `font-size` using the size reference table above.

6. **Group multiple badges** — Wrap them in a flex container:
   ```html
   <div style="display: flex; gap: 8px; flex-wrap: wrap;">
     <!-- badges go here -->
   </div>
   ```
</Steps>

## When to Use Badges

| Use Case | Example | Recommended Variant |
|---|---|---|
| Status indicators | New, Updated, Active | Semantic (Success, Info, Warning) |
| Category labels | Department names, subject areas | Primary or Secondary |
| Dates and timestamps | December 2025, Q3 | Small, Light Blue |
| Tags and keywords | Technology, Arts, STEM | Primary or Secondary group |
| Counts and metrics | 12 Items, 3 New | Small, Navy or Gold |

## Accessibility

<Aside type="note" title="Accessibility Features">
- Semantic `<span>` elements for proper inline display
- All color combinations meet **WCAG AA** contrast standards for readability
- Font weight `600` ensures clarity at all sizes and screen densities
- No JavaScript required — works with all assistive technologies
- `inline-flex` layout ensures correct alignment within text and headings
- `white-space: nowrap` prevents awkward mid-word wrapping inside badges
</Aside>

## Best Practices

<Aside type="tip" title="Do's">
- Keep badge text short — **1 to 3 words** maximum
- Use **consistent colors** for similar types of information across a page
- Choose **appropriate size** based on the surrounding content hierarchy
- **Group related badges** together with consistent spacing (`gap: 8px`)
- Use **semantic colors** (Success, Warning, Error) for status indicators
- Use **outline style** for secondary or supplemental information
</Aside>

<Aside type="caution" title="Don'ts">
- Avoid using **too many badges** in one area — it causes visual clutter
- Don't use badges for **long text** — use the Alert component instead
- Avoid mixing **too many different colors** in a single badge group
- Don't use **low-contrast** color combinations
- Avoid using badges as **clickable buttons** — use a proper Button component
- Don't override the `inline-flex` display property
</Aside>
